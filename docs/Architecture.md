# 🚀 ESP32-S3 工业级蜂窝物联网架构技术文档（v2.0 演进版）

**项目名称：** 基于 FreeRTOS 与 4G 蜂窝网络的高可用 IoT 终端及云端中台架构
**更新日期：** 2026-04
**核心定位：** 生产级、全链路、高可用

## 💡 v2.0 重构亮点 (架构代差升级)
本次更新彻底摒弃了局域网与裸机单线程的局限，全面拥抱真正的工业物联网 (IIoT) 标准：
* **广域网脱网**：使用 4G Cat.1 模块替代 Wi-Fi，实现全天候公网直连。
* **双核并发引擎**：引入 FreeRTOS 操作系统，压榨 ESP32-S3 双核算力，实现网络与硬件中断的物理隔离。
* **金融级传输**：废弃明文传输，全面启用 MQTTS (基于 MbedTLS 硬件加速) 双向加密。
* **极端容灾防御**：引入独立硬件看门狗 (WDT) 防死机，以及 LittleFS 本地离线缓存防数据丢失。

## 🏗️ v2.0 系统全景架构

### 1. 硬件通信拓扑

设备端采用主从芯片架构，将计算与通信解耦：
* **主控核心 (ESP32-S3)**：负责传感器采样、数据 JSON 序列化、LittleFS 读写与 TLS 加密解密。
* **通信网关 (4G Cat.1 模块)**：通过 UART 串口与主控相连，接收 AT 指令或运行 PPP 拨号栈，充当纯粹的公网透明传输通道。

### 2. 软件调度架构 (FreeRTOS)

为了保证实时响应，系统将原本的单线程拆分为双核隔离任务：
* **Core 0 (网络核心)**：绑定 `Task_Network`，专门处理高延迟的 AT 指令解析、MQTTS 心跳维护与云端指令接收。
* **Core 1 (硬件核心)**：绑定 `Task_Sensor` 和 `Task_Control`，专注于毫秒级的硬件中断响应、内部温度读取与看门狗喂狗。
* **进程间通信 (IPC)**：两个核心之间严禁全局变量乱指，统一使用 FreeRTOS 的 `Queue` (消息队列) 传递 JSON 数据结构。

### 3. 全栈云端链路
设备采集数据后，经由基站进入云端，与后端系统完成交互：
* **接入层**：企业级 MQTT Broker 集群 (支持 8883 端口 TLS 卸载)。
* **业务层**：Java Spring Boot 中台，集成 `spring-integration-mqtt`，执行通配符 `+/state` 订阅。
* **持久层**：MySQL 存储时序遥测数据。
* **展示层**：Vue 或 JavaFX 提供数字孪生可视化大屏。

## 🛡️ 工业级容灾与防坑指南

### 防御 1：LittleFS 断网数据“防漏斗”机制
工业设备经常部署在信号盲区（如地下室）。断网时如果继续调用 `publish()`，数据将永久丢失。
* **解决逻辑**：每次采集到温度或状态，先将 JSON 字符串追加写入本地 Flash 的 LittleFS 文件系统中（例如 `/cache/telemetry.txt`）。
* **补发机制**：`Task_Network` 检测到 4G 网络恢复后，优先读取缓存文件，将积压的数据逐条通过 MQTT 上报，并清空本地文件。

### 防御 2：硬件看门狗 (Task Watchdog Timer)
软件写的重连机制再好，一旦遇到宇宙射线导致的寄存器翻转或指针越界，系统死机将无力回天。
* **解决逻辑**：引入 ESP-IDF 底层的 TWDT。在系统初始化时启动看门狗，设置超时阈值为 15 秒。
* **喂狗动作**：在核心业务的死循环中高频调用 `esp_task_wdt_reset()`。一旦 4G 模块无响应卡死，程序挂起，看门狗将在 15 秒后切断系统电源并完成硬件级硬重启。

### 防御 3：MbedTLS 内存爆缸预警
启用 MQTTS (TLS 1.2) 会瞬间吃掉大量的动态内存 (Heap) 用于 RSA 握手计算。
* **解决逻辑**：必须利用 ESP32-S3 的 PSRAM（外部伪静态随机存取存储器）。在 PlatformIO 中配置开启 PSRAM，并将网络任务的堆栈分配在 PSRAM 中，防止内存溢出导致 Core Panic。

## 🛠️ 核心开发步骤建议 (v2.0 路线图)

这是一套庞大且严密的工程体系，强烈建议按以下顺序分步攻克：

1. **操作系统改造**：不动网络，先把目前的单线程代码重构进 FreeRTOS 的 `xTaskCreatePinnedToCore` 中，跑通双核通信队列。
2. **硬件换血通信**：购入 4G Cat.1 模块，用两根杜邦线接通 ESP32 的串口，手搓 AT 指令集解析器，实现模块连上基站并获取 IP。
3. **安全链路打通**：在 MQTT 客户端库中导入根证书，将端口改为 8883，完成第一次安全的 MQTTS 连接。
4. **容灾加固**：挂载 LittleFS，模拟拔掉 4G 模块天线，测试数据的本地缓存与断网续传能力。
5. **后端中台承接**：启动 Java Spring Boot 项目，完成数据库建表与实时流数据的订阅写入。
